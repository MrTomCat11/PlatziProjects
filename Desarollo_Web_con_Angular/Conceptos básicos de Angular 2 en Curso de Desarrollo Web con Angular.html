@Input

Los componentes son la base de todo proyecto hecho en Angular y una de las cosas más importantes es cómo compartir información entre ellos.

@Input es un componente de entrada de datos, y la forma en que lo declaramos es con el decorator “@input” al momento de exportar la clase:

import { Component, Input } from '@angular/core';

@Component({
    selector: 'input-component',
    templateUrl: 'input.component.html',
    styleUrls: ['input.component.css']
})
export class InputComponent{

    @Input() voto:string;                // Se usa el decorator @Input() para declarar que tendremos un input del tipo string de nombre voto

    constructor(){
        console.log("estoy entrando");  // Hacemos un console.log() para asegurar que entramos a un componente

    }
    
}

¿Entonces dónde se encuentras ese input que creamos?, se en efecto un selector del componente llamado voto

<h1>
  {{title}}
</h1>
<div *ngFor="let voto of votos">
  <input-component voto="{{voto.title}}"></input-component>
</div>

Output

Con Output, podemos pasar datos de componentes hijos a componentes padres. ¿Cómo hacemos esto?, lo hacemos mediante eventos.

Vamos al HTML de nuestro componente hijo input.component para configurar el evento:

<span (click)="votar()">{{voto}}<span>

En este caso, vamos a escuchar el evento “click” que va a activar la función votar (), declaremos esta función en la funcionalidad del componente:

import { Component, Input, Output, EventEmitter } from '@angular/core';  // Importamos Output y EventEmiiter 

@Component({
    selector: 'input-component',
    templateUrl: 'input.component.html',
    styleUrls: ['input.component.css']
})
export class InputComponent{

    @Input() voto:string;

    @Output() addVoto = new EventEmitter<string>();       // Indicamos que nuestro componente tendrá un eventemitter que regresa un string 

    constructor(){
        console.log("estoy entrando");
    }

    votar():void{
        this.addVoto.emit(this.voto);                      // Entonces lo se que va a emitir es el string que tenga voto 
    }

}

Si vamos ahora al HTML del componente padre, vamos a configurar un nuevo atributo que describe el @output:

<h1>
  {{title}}
</h1>
<div *ngFor="let voto of votos">
  <input-component voto="{{voto.title}}" (addVoto)="addVoto($event)"></input-component>
</div>
<h3>{{votacion}}</h3>

(addVoto) es el nombre que le asignamos a nuestro @output en el componente hijo

“addVoto($event)” es el nombre de la función que vamos a activar desde el componente padre:

import { Component } from '@angular/core';

import { InputComponent } from './input/input.component';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'app works!';
  votacion = '';

  votos = [
    {title : 'Opción 1'},
    {title : 'Opción 2'},
    {title : 'Opción 3'},
    {title : 'Opción 4'}
  ];

  addVoto(response: string) {
    this.votacion = "Usted eligió: " + response;      // Esta es la función que se va a ejecutar en el componente padre 
  }

}

Y si recordamos el HTML del componente padre, se hace bind de la variable votación, por eso cambia este h3 cada vez que hacemos click en cualquiera de las opciones:

<h3>{{votacion}}</h3>

Pipes " | "

Los pipes son operadores que nos ayudan a transformar los datos que vienen del componente a nuestra plantilla. Por defecto, angular cuenta con algunos ya definidos, por ejemplo: DatePipe, UpperCasePipe, LowerCasePipe, CurrencyPipe, and PercentPipe ( que son muy descriptivas sus funcionalidades por su nombre), sin embargo tambien tenemos la oportunidad de crear nuestros pipes personalizados.

ejemplo : Vamos a crear un Pipe que nos permita hacer una conversión de una moneda a otra ( currency )

Lo primero que vamos a hacer es importar Pipe y PipeTransform ( esto no será un componente):

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({name: 'conversorPipe'}) // En el annotation vamos a declarar el nombre de nuestro pipe

export class ConversorPipe implements PipeTransform {    // vamos a implementar un PipeTransform
  transform(value: number, exponent: string): number {
    let exp = parseFloat(exponent);                                 // vamos a recibir un numero y un string, el string lo vamos a convertir en numero flotante y de inmediato hacer la multiplicación 
    return value * (isNaN(exp) ? 1 : exp);			     // sin embargo vamos a validad que sea un numero o dar un valor de 1 en otro caso. 
  }
}

Ahora en la vista del componente padre vamos a conectar los inputs a nuestro modelo usando ngModel, y hacer uso de ellos en la lógica del componente, además, de invocar nuestro pipe:

<h2>Conversor</h2>
<div>Cantidad: <input [(ngModel)]="cantidad"></div>
<div>1 dolar es : <input [(ngModel)]="factor"></div>
<p>
  Oh: {{cantidad | conversorPipe: factor}}
</p>

Nos aseguramos que las variables que usamos en la plantilla con ngModel estén declaradas de igual manera en el componente, es recomendado iniciarlas con cierto valor.

import { Component } from '@angular/core';

import { InputComponent } from './input/input.component';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'app works!';
  votacion = '';

  votos = [
    {title : 'Opción 1'},
    {title : 'Opción 2'},
    {title : 'Opción 3'},
    {title : 'Opción 4'}
  ];

  addVoto(response: string) {
    this.votacion = "Usted eligio: " + response;
  }

  cantidad = 5;
  factor = 1;             // aquí están las variables que mandamos a llamar en nuestra plantilla usando ngModel
}

¿Qué otras funcionalidades podemos tener con Angular CLI?

Volvamos a nuestro Angular CLI, que si recordamos nos ayudo a construir nuestro primer proyecto. Entre los comandos que nos ayudan a crear elementos de manera más fácil:

ng g component my-new-component --> Crea un nuevo componente
ng g directive my-new-directive --> Crea una nueva directiva
ng g pipe my-new-pipe --> Crea un nuevo Pipe
ng g service my-new-service --> Crea un nuevo servicio
ng g class my-new-class --> Crea una nueva clase
ng g interface my-new-interface --> Crea una nueva interface

Pero existen otros que nos ayudan con el flujo de todo el proyecto, por ejemplo:

ng build --> Hace todo el build para llevar el build a producción
ng serve --> Levanta un servidor local con el proyecto
ng serve --ssl true --> Levanta un servidor local HTTPS para pruebas de servicios que solo soporten dicho protocolo
ng test --> Corre todo los test unitarios

Y cada día se agregan más herramientas, puedes consultarlas en:

https://cli.angular.io/reference.pdf

Directivas

Las directivas nos ayudan a cambiar la apariencia o el comportamiento de un elemento del DOM.

Por ejemplo, quiero destacar un elemento del DOM con el color amarillo, para ello creo una directiva:

import { Directive, ElementRef, Input } from '@angular/core';   // Importo la directiva, ademas de ElementRef, Input que nos ayudan a determinar que elemento estamos en el DOM.

@Directive({ selector: '[myHighlight]' })       // el selector define el nombre de la directiva
export class HighlightDirective {           
    constructor(el: ElementRef) {                 // se recibe como parámetro el elemento 
       el.nativeElement.style.backgroundColor = 'yellow';    // Aquí cambio de inmediato el atributo, en este caso el color de background. 
    }
}

Entonces, en las vistas de nuestros componentes ya podemos incluir estas directivas para modificar su comportamiento, por ejemplo:

<p myHighlight>Ahora soy Amarillo</p>

Inclusive, un elemento puede contener muchas directivas:

<p myHighlight gigant>Ahora soy Amarillo y grande</p>

Inyección de Dependencias

En angular es muy importante entender que un servicio puede ser inyectado en cualquier componente, mediante un patrón de software llamado singleton, en otras palabras, al iniciar nuestra aplicación todos los providers se van a instanciar en memoria y los vamos a poder utilizar en todos lados, y por ejemplo podemos pasar parametros más completos entre componentes o inclusive componentes mismos.

Por ejemplo, si tenemos una conexión a base de datos, podemos hacer todas las consultas desde un servicio y ese servicio inyectarlo como dependencia en un componente, así, podemos disponer de dichas consultas desde cualquier parte de la aplicación, no solo el componente.

Pongamos un ejemplo de inyección de dependencias

Tenemos un arreglo que esta creado por:

export var TICKETS:Array<any> = [
    {'id': 1, 'titulo': 'no me funciona la impresora', 'estado': 'in progress'},
    {'id': 2, 'titulo': 'no me funciona la computadora', 'estado': 'finish'},
    {'id': 3, 'titulo': 'no me funciona el celular', 'estado': 'in progress'},
    {'id': 4, 'titulo': 'no me funciona una lampara', 'estado': 'really'}   
];

Vamos a crear un servicio que nos ayude a obtener información, en este caso, importamos el arreglo desde ./mocks/tickets.mock.

import { Injectable } from '@angular/core';
import { TICKETS } from './mocks/tickets.mock';

@Injectable()  // declaramos que es un Injectable
export class TicketService{

    miVariableTicketGlobal = "soy una variable"; // podemos crear variables globales en cualquier componente

    getTickets(){
        return TICKETS;     // regresamos todos los tickets 
    }

    getVariableGlobal():string{
        return this.miVariableTicketGlobal;    
    }

}

Luego, en nuestro componente, hacemos la declaración de este servicio en el constructor:

import { Component } from '@angular/core';

import { InputComponent } from './input/input.component';

import {TicketService} from './services/ticket.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'app works!';
  votacion = '';

  votos = [
    {title : 'Opción 1'},
    {title : 'Opción 2'},
    {title : 'Opción 3'},
    {title : 'Opción 4'}
  ];

  tickets:any;

  constructor(private ticketService :TicketService ){        // Al instancias declaro que voy a injectar un            		                // TicketService
    this.tickets = ticketService.getTickets();                     // Aquí puedo hacer el llamado del metodo para //obtener todos los ticktets     
    console.log(this.tickets);              
  }

  addVoto(response: string) {
    this.votacion = "Usted eligio: " + response;
  }

  cantidad = 5;
  factor = 1;
}

Ya en la vista, puedo hacer uso de los datos de los tickets, recorriendo los elementos por un *ngFor,

<h1>
  {{title}}
</h1>
<div *ngFor="let voto of votos">
  <input-component voto="{{voto.title}}" (addVoto)="addVoto($event)"></input-component>
</div>
<h3>{{votacion}}</h3>


<h2>Conversor</h2>
<div>Cantidad: <input [(ngModel)]="cantidad"></div>
<div>1 dolar es : <input [(ngModel)]="factor"></div>
<p myHighlight gigant>
  Oh: {{cantidad | conversorPipe: factor}}
</p>

<ul>
  <li *ngFor="let ticket of tickets" gigant>
    <b>Ticket N: {{ticket.id}}</b> - {{ticket.titulo}} 
    <span myHighlight>[ {{ticket.estado}} ]</span>
  </li>
</ul>

Ya por último en nuestro app.module.ts, no vamos a declararlo como un NgModule, si no como un provider:

import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { FormsModule } from '@angular/forms';
import { HttpModule } from '@angular/http';

import { AppComponent } from './app.component';


//input 
import {InputComponent} from './input/input.component';

//pipes
import { ConversorPipe } from './pipes/conversor.pipe';

//directivas
import { HighlightDirective } from './directives/highlight.directive';
import { GigantDirective } from './directives/gigant.directive';

//DI services
import { TicketService } from './services/ticket.service';

@NgModule({
  declarations: [
    AppComponent,
    InputComponent,
    ConversorPipe,
    HighlightDirective,
    GigantDirective
  ],
  imports: [
    BrowserModule,
    FormsModule,
    HttpModule
  ],
  providers: [TicketService],
  bootstrap: [AppComponent]
})
export class AppModule { }
